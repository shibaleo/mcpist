## 詳細

| 項目 | 内容 |
|------|------|
| トリガー | トークン保存・取得・削除時 |
| 操作 | ユーザーID単位でトークン所有権を管理（関連付け/参照/解除） |

### 期待する動作

- トークン保存時は「ユーザーID × サービス」の単位で関連付けを作成する  
  - 同一ユーザー・同一サービスのトークンは常に単一の関連付けのみを持つ  
  - 既存の関連付けがある場合は上書き（最新化）される
- トークン取得時は「ユーザーID × サービス」で検索し、所有者が一致する場合のみ返す  
  - 取得結果は「存在する/しない」を明確に区別する
- トークン削除時は「ユーザーID × サービス」の関連付けを解除する  
  - 解除後は取得しても存在しない扱いになる
- 他ユーザーのトークンにはアクセスできない（ユーザーID境界の強制）

### データフロー

| 方向        | 内容                  |
| --------- | ------------------- |
| DST → TVL | ユーザーIDによるトークン所有者の特定 |


### 期待する表現（結果の扱い）

- トークン取得の結果は以下を区別して扱う  
  - **存在**: ユーザーに紐づくトークンが見つかった  
  - **不存在**: 該当ユーザー/サービスのトークンが存在しない  
  - **不一致**: ユーザーIDが一致せず取得できない（アクセス不可）

### 制約

- 管理対象はユーザーIDとの紐づけのみ（サービス定義・権限・更新履歴等は本仕様の対象外）
