## 詳細

| 項目 | 内容 |
|------|------|
| 方式 | Webhook |
| プロトコル | HTTPS |
| 認証 | 署名検証（Webhook Secret） |
| データ形式 | JSON |

**Webhookエンドポイント:** `https://api.mcpist.app/webhooks/psp`

### 期待する動作

- 決済完了イベントを受信した場合、有料クレジット残高を加算する
- セッション期限切れ等、非課金イベントは状態を変更しない
- 署名検証に成功したイベントのみ処理する
- 同一イベントIDの重複処理は行わない
- イベント順序は保証されないため、順不同でも整合するよう処理する

### 通知されるイベント（期待）

| イベント | 説明 | 期待する処理 |
|----------|------|--------------|
| checkout.session.completed | 決済完了 | 有料クレジット残高加算 |
| checkout.session.expired | セッション期限切れ | 変更なし |

**注:** PSPは有料クレジットのみを扱う。無料クレジットはシステム内部で管理される

### 期待する表現（結果の扱い）

- 受信結果
  - **成功**: 署名検証済みイベントとして受理し、必要な更新が完了した
  - **失敗**: 署名不正/形式不正/整合性エラー等で処理できない
  - **無視**: 非課金イベント、または重複イベントとして処理対象外

### 制約

- 管理対象は有料クレジットの加算のみ（返金・チャージバック・課金状態の詳細管理は本仕様の対象外）
- 実装は未完了（本仕様は将来実装の期待動作を定義）
