- テスト: 生成された JWT をデコードし `key_id` が含まれることを確認

---

## Phase 2: API キー認証時に DB 照合を追加 (MCPIST-2 / S11-002)

- `ogenserver/security.go`: API キー認証パスで以下を追加:
  1. JWT から `key_id` を抽出
  2. `db.GetAPIKeyByID(key_id)` で存在確認
  3. 存在しない or `revoked_at IS NOT NULL` → 401
- パフォーマンス: 毎リクエスト DB 照合になるので、必要に応じてインメモリキャッシュ (TTL 5 分) を検討

---

## Phase 3: Worker 側にも DB 照合を追加 (MCPIST-1 / S11-003)

- `apps/worker/src/auth.ts`: Worker が JWT を検証するフローに key_id 照合を追加
- Worker は DB に直接アクセスできないので、Go Server に照合 API を呼ぶか、JWKS に revocation 情報を含めるか、方式を決める必要がある
- 選択肢:
  - A: Worker → Go Server に `GET /v1/internal/keys/{key_id}/status` を追加
  - B: Worker の JWT 検証時に Go Server の既存 API で照合
  - C: Worker は署名検証のみ、Go Server 側で全 API キーアクセスをゲートする

### S11-034: expires_at 形式統一 ✅

**現状調査:**
- OAuth callback ルートで 2 つの形式が混在:
  - **Unix timestamp (秒):** Google, Microsoft, Atlassian, Notion → `Math.floor(Date.now() / 1000) + tokenData.expires_in`
  - **ISO 8601 文字列:** Airtable, Asana, TickTick, Dropbox → `new Date(Date.now() + tokenData.expires_in * 1000).toISOString()`
  - **null (期限なし):** GitHub, Todoist, Trello
- Go Server 側 (`broker/token.go`) は `FlexibleTime` 型で両方をパースできるが、統一した方がデータ整合性が高い

**結果:** 4 プロバイダ (Airtable, Asana, TickTick, Dropbox) の callback を Unix timestamp (秒) に統一。

### セキュリティ再監査: SSRF 対策強化

**背景:** Phase 2 完了後の再監査で、`credentials/validate` エンドポイントが認証付きでも SSRF リスクを持つことが判明。`domain` / `base_url` パラメータで任意ホストへリクエストを送信可能。

**対策方針:** 現実的な攻撃シナリオに集中。KV/Redis が必要な OAuth state nonce バインディングは見送り。

**実装内容:**

| バリデータ | 対策 |
|---|---|
| Jira, Confluence | `*.atlassian.net` ドメイン許可リスト |
| Grafana | https 必須 + プライベート IP / localhost / メタデータ IP (`169.254.169.254`) 拒否 |
| 全 7 バリデータ | `AbortSignal.timeout(5000)` + `redirect: 'error'` |

**変更ファイル:**
- `validators/jira.ts` — allowlist + timeout + redirect 拒否
- `validators/confluence.ts` — allowlist + timeout + redirect 拒否
- `validators/grafana.ts` — URL 検証 + timeout + redirect 拒否
- `validators/github.ts` — timeout + redirect 拒否
- `validators/notion.ts` — timeout + redirect 拒否
- `validators/supabase.ts` — timeout + redirect 拒否
- `validators/trello.ts` — timeout + redirect 拒否
