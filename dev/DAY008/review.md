## 実装の進め方

各コンポーネント/機能ごとに以下のサイクルを回す。

```
1. 実装範囲を決定
       │
       ▼
2. 大まかな仕様書に沿ったプロトタイプを作成
       │
       ▼
3. プロトタイプに対してテストを作成
       │
       ▼
4. 詳細仕様を作成
       │
       ▼
5. 詳細テスト仕様を作成
       │
       ▼
6. 詳細仕様に合うようにプロトタイプをリファクタリング
       │
       ▼
7. 詳細テスト仕様を確定
       │
       ▼
8. 詳細仕様を確定
       │
       ▼
9. 全体の仕様と整合性が取れていることを確認
```

**ポイント:**
- 仕様を先に完璧にしない（プロトタイプで検証してから詳細化）
- テストはプロトタイプ作成後に作成
- 詳細仕様は実装と並行して作成
- 各サイクル完了時に全体仕様との整合性を確認

### 改善点

- 単体テストは詳細設計なしには書けない → TBDで正解
- 出力形式（TOON/Markdown）は仕様書ではなくモジュールの責務 → 削除で正解

### 学び

- 仕様書は実装状態（対策済み/未対応）を記載しない
- 仕様書は要件を定義し、設計書は実現方法を定義する
